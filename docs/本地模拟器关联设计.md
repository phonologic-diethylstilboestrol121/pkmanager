# 本地模拟器启动框架 — 设计文档

> **日期**: 2026-06-06（更新：Azahar 源码分析完成）
> **源码参考**: `sdk/azahar/src/` — 已分析 CLI 参数、存档路径、标题管理
> **目标**: 搭建一个通用框架，让 pkmanager 能启动本地模拟器（DeSmuME NDS、Azahar 3DS），并管理存档的双向同步。
> **原则**: 简单、通用、可扩展。两个模拟器共用同一套基础设施。

---

## 1. 整体思路

```
用户在浏览器点击「启动 DeSmuME」
          │
          ▼
┌─────────────────────────────────────────────────────┐
│  前端 (React)                                       │
│  SettingsPage: 配置 Azahar/DeSmuME 的可执行文件路径 │
│  SavesPage:   存档列表每一行多了「本机启动」按钮      │
│  Dashboard:   游戏卡片检测配置状态                   │
└──────────────┬──────────────────────────────────────┘
               │ POST /api/Emulator/launch-local
               ▼
┌─────────────────────────────────────────────────────┐
│  后端 (ASP.NET Core)                                │
│                                                     │
│  ① 读取配置 (user_settings 表)                       │
│  ② 复制存档 → 模拟器目录                             │
│  ③ 返回启动包（exe/save/rom/syncToken）给浏览器       │
│  ④ 本地协议启动器 / 回退脚本启动模拟器               │
│                                                     │
│  模拟器关闭后:                                       │
│  ⑤ 启动器从本机模拟器目录读回存档                    │
│  ⑥ 二进制 POST 回 pkmanager，成功后恢复本机旧存档    │
└─────────────────────────────────────────────────────┘
```

**核心思想：后端负责生成启动包和同步 token；真正启动模拟器的是浏览器所在机器的协议启动器 / 回退脚本。NDS 传 ROM 路径启动，3DS 直接启动 Azahar 已安装标题。**

## 2. 数据库设计

用 key-value 模式 + device_id 维度。好处：新增模拟器无需改表结构，同一用户不同电脑各自独立配置。

```sql
CREATE TABLE IF NOT EXISTS user_settings (
    user_id     UUID        NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    device_id   UUID        NOT NULL,   -- 前端生成的设备指纹
    key         VARCHAR(64) NOT NULL,   -- 如 'azahar.exe_path', 'desmume.exe_path'
    value       TEXT        NOT NULL,   -- JSON 或纯字符串
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (user_id, device_id, key)
);
```

### device_id 是什么

```
浏览器首次访问 pkmanager
  → 检查 localStorage 是否有 'pkmanager_device_id'
  → 没有 → 生成一个新 UUID (crypto.randomUUID())
  → 存入 localStorage，持久化
  → 后续所有 API 请求自动带上 X-Device-Id 请求头
```

```
同一用户 + 电脑A: device_id = "3f2a-..." → azahar.exe_path = /usr/bin/azahar
同一用户 + 电脑B: device_id = "8e1b-..." → azahar.exe_path = C:\Tools\azahar.exe
```

- 换电脑 → 新 deviceId → 设置页是空的，重新配一次即可
- 同一电脑 → 同一 deviceId → 配置自动匹配
- 用户完全无感，不需要记住任何东西

### 各模拟器需要的 key

| key | 示例值 | 说明 |
|-----|--------|------|
| `azahar.exe_path` | `/usr/bin/azahar` | Azahar 可执行文件路径 |
| `azahar.data_dir` | `~/.local/share/azahar-emu` | Azahar 用户数据目录（含 SDMC） |
| `desmume.exe_path` | `/usr/bin/desmume` | DeSmuME 可执行文件路径（CLI 版本） |
| `desmume.save_dir` | `~/.config/desmume` | （可选）存档目录，默认自动检测 XDG 路径 |

## 3. device_id 传递链路

```
前端 axios 请求拦截器:
  config.headers['X-Device-Id'] = localStorage.getItem('pkmanager_device_id')

后端从请求头读取:
  Request.Headers["X-Device-Id"] → Guid deviceId

SettingsService 查询/写入:
  WHERE user_id = @UserId AND device_id = @DeviceId
```

初始化时机：`main.tsx` 中检查 `localStorage`，不存在就 `crypto.randomUUID()` 生成。

## 4. 后端新增

### 4.1 SettingsController（配置管理）

```
GET  /api/settings/emulators        → 返回当前 device 的模拟器配置
PUT  /api/settings/emulators        → 保存配置
POST /api/settings/emulators/test   → 测试模拟器是否能启动（启动后立即退出，返回版本号）
```

### 4.2 EmulatorController 扩展（启动）

```
POST /api/Emulator/launch-local/{saveFileId}
```

流程：
```
1. 查存档 → 获取 gameVersion + generation
2. 查配置 → user_settings 表 → 获取模拟器路径
3. 复制存档:
   - NDS: save.sav → {desmume.save_dir}/{rom_name}.dsv（需找到 ROM 文件）
   - 3DS: save.sav → title/.../data/00000001/main（CIA 已安装，无需 ROM）
4. 自动备份 (标签: "启动本地模拟器前")
5. 返回启动包给前端：
   - `exePath`, `romPath`, `emuSavePath`, `saveDataBase64`, `saveFileId`, `syncToken`
6. 浏览器优先调起 `pkmanager://launch/{token}` 协议；无协议时下载本地脚本执行
```

### 4.3 EmulatorController — 存档回传

```
POST /api/Emulator/sync-save/{saveFileId}?token={syncToken}
```

由本地启动器在模拟器退出后直接上传二进制存档，服务器写入 pkmanager。

### 4.4 EmulatorController — 进程状态

协议启动路径下由本地启动器等待进程退出，不再依赖浏览器轮询 PID 判断生命周期。

## 6. 存档映射

### NDS (DeSmuME)

**源码参考**: `sdk/desmume/desmume/src/path.cpp`, `mc.cpp`, `frontend/posix/cli/main.cpp`

**用户数据目录** (默认 `~/.config/desmume/`):
```
~/.config/desmume/
  ├── <ROM名>.dsv      ← 存档文件（raw save，PKHeX 直接兼容）
  ├── <ROM名>.dsN      ← 即时存档 (ds0~ds9)
  ├── <ROM名>.dct      ← 金手指
  ├── config           ← 按键映射
  └── config.cfg       ← 模拟器设置
```

**存档路径规则** (源码 `path.cpp:LoadModulePath()` + `mc.cpp:BackupDevice()`):
DeSmuME 将 `.dsv` 存放在 `~/.config/desmume/` 下，文件名 = ROM 文件名（去扩展名）+ `.dsv`。例如 ROM 文件为 `口袋妖怪：珍珠.nds`，则存档为 `~/.config/desmume/口袋妖怪：珍珠.dsv`。

注意：存档目录**不可通过 CLI 参数指定**，固定为 XDG 配置目录。如需自定义，可通过设置 `XDG_CONFIG_HOME` 环境变量实现。

**CLI 启动** (源码 `cli/main.cpp`):
```bash
desmume-cli [options] <rom_file.nds>
```
ROM 路径为必选位置参数。常用选项：`--load-slot N`（加载即时存档）、`--disable-limiter`（关闭限帧）。

**存档迁移**: 如果 DeSmuME 找不到 `.dsv`，会自动尝试 `.sav` 并转换为 `.dsv`（`mc.cpp` BackupDevice 构造函数）。.dsv 本质就是 raw save，PKHeX.Core 可直接读取。

**存档同步流程**:
```
pkmanager: data/saves/{userId}/{saveFileId}/save.sav
                │  启动前: 复制为 ~/.config/desmume/<ROM名>.dsv
                ▼
DeSmuME:   ~/.config/desmume/<ROM名>.dsv
                │  关闭后: 复制回 pkmanager
                ▼
pkmanager: data/saves/{userId}/{saveFileId}/save.sav  (更新)
```

### 3DS (Azahar)

**用户数据目录** (默认 `~/.local/share/azahar-emu/`):
```
~/.local/share/azahar-emu/
  ├── sdmc/          ← 虚拟 SD 卡（CIA 安装位置 + 存档）
  ├── nand/          ← NAND 文件系统
  ├── config/        ← qt-config.ini
  └── ...
```

**存档路径** (源码 `core/file_sys/archive_source_sd_savedata.cpp`):
```
{azahar.data_dir}/sdmc/Nintendo 3DS/
  00000000000000000000000000000000/00000000000000000000000000000000/
  title/00040000/{titleId后8位}/data/00000001/main
```
其中 `00040000` 是零售游戏固定前缀，`{titleId后8位}` 为游戏特有的 8 位 hex（如 X=00055D00）。

**启动方式** (源码 `citra_qt.cpp` `BootGame()`):
Azahar 支持 CLI 传入文件路径直接启动游戏，无需用户手动选择：
```bash
azahar "{sdmc}/Nintendo 3DS/.../title/00040000/{tid_low}/content/00000000.app"
```
传入 title 目录下 `content/` 中的 `.app` 内容文件即可。

**存档同步流程**:
```
pkmanager: data/saves/{userId}/{saveFileId}/save.sav
                │  启动前复制
                ▼
模拟器:    {azahar.data_dir}/sdmc/Nintendo 3DS/
           .../title/00040000/{titleId后8位}/
           data/00000001/main
                │  关闭后复制回来
                ▼
pkmanager: data/saves/{userId}/{saveFileId}/save.sav
```

## 8. 安全考量

- 启动器使用的 `exePath` / `romPath` / `emuSavePath` 全部来自服务器生成的启动包，**不是浏览器直接拼接任意路径**
- 模拟器路径通过 `File.Exists()` 验证后才启动
- 存档操作前自动备份（5 槽位轮转），任何情况下可恢复
- `[Authorize]` 保护所有端点
